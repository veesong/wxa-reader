1. 适用范围与当前版本说明

本政策描述的是当前已发布版本的实际行为，而不是未来可能上线的功能计划。

当前版本没有接入广告 SDK、统计分析 SDK、崩溃上报 SDK，也没有账号注册、登录、云同步或消息推送能力。

当前版本也不会要求你提交姓名、手机号、身份证号等实名信息。

如果后续版本新增广告、用户登录、云同步或其他会影响数据处理方式的能力，我会在对应版本发布前先更新本隐私政策，并按实际上线的功能重新说明数据收集、使用和共享情况。

2. 本地存储的数据

2.1 阅读历史

  • 本地保存文章链接、文章标题、阅读时间，以及用于移除页面广告块的清理规则。
  • 用途是标记已读状态，并在“历史”页面展示最近读过的文章。
  • 数据只保存在设备本地，最多保留最近 200 条。

2.2 收藏记录

  • 本地保存文章链接、文章标题、收藏时间，以及用于移除页面广告块的清理规则。
  • 用途是支持“收藏 / 取消收藏”和收藏列表展示。
  • 数据只保存在设备本地，直到你主动取消收藏、清除应用数据或卸载应用。

2.3 阅读偏好

  • 当前会保存字号大小等界面偏好设置。
  • 用途是让应用在下次打开时保持你的阅读习惯。
  • 这些偏好仅保存在设备本地，不用于跨应用跟踪。

2.4 更新提醒状态

  • 本地保存最近检查更新时间、最近一次检查是否失败、最近提示过的版本号。
  • 用途是避免重复弹出同一版本提示，并在网络失败后按较短间隔重试。
  • 这些信息仅用于版本提醒控制，不包含你的真实身份信息。

3. 网络访问与第三方服务

3.1 项目内容分发服务

  • 应用会从项目内容分发地址拉取文章列表，用于展示最近可读内容。
  • 列表数据在传输前已做加密封装，应用在本地解密后展示。
  • 与常见网络服务一样，这类请求可能在服务端产生常规访问日志，例如 IP 地址、请求时间和 User-Agent。

3.2 文章原始来源

  • 当你打开某篇文章时，应用会直接请求该文章的原始链接，常见来源包括微信公众号页面。
  • 请求目的是抓取正文并在本地做清理显示，而不是把你的阅读历史上传给开发者做画像。
  • 原始内容站点会按照其自身的隐私政策处理访问日志和相关数据。

3.3 GitHub Releases

  • 应用会访问 GitHub 的 latest releases 接口，用于判断是否有新版本可下载。
  • 该过程不会要求你登录 GitHub，也不会读取你的 GitHub 账号资料。
  • 访问 GitHub API 时，同样可能产生 GitHub 侧的常规访问日志。

3.4 当前不会进行的网络处理

  • 当前版本不会把阅读历史、收藏记录或字号偏好同步到开发者后台。
  • 当前版本不会将设备通讯录、照片、定位、麦克风或摄像头信息上传到服务器。
  • 当前版本不会将你的数据出售、出租，也不会用于广告定向。

4. 权限、共享与保留

4.1 应用权限

当前应用仅申请网络权限，用于获取文章列表、读取文章原文和检查新版本。

4.2 数据共享

数据只会在你主动触发的网络请求中发送到内容源站点、GitHub 或项目内容分发服务。当前版本不会额外共享给广告平台、统计平台或第三方登录平台。

4.3 数据保留与删除

你可以通过清除应用数据或卸载应用，删除本地保存的历史、收藏、阅读偏好和更新提醒状态。

5. 儿童隐私

本应用面向普通内容阅读场景，不以 13 岁以下儿童为目标用户，也不会主动要求儿童提供身份信息。

如果监护人认为儿童向本应用提供了不应收集的信息，可通过本页提供的联系方式联系开发者处理。

6. 政策更新与联系

如果后续版本新增广告、用户登录、云同步、消息推送、统计分析等能力，本页会先更新对应说明，再随版本一起发布。

如对本政策有疑问，或希望反馈数据相关问题，可通过 GitHub Issues 联系开发者。
