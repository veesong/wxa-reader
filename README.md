# 公众号阅读

这是一个围绕我个人日常阅读习惯搭建的公众号阅读项目。

平时看到值得保存或值得反复阅读的公众号文章，我会把它们投稿到这个系统里。项目会经过抓取、整理、筛选和审核，最后以 RSS 的形式输出，方便在更安静的阅读环境里继续看。

需要说明的是：RSS 里的文章都是我自己日常阅读之后筛选出来的，内容口味取决于我个人，可能包含你不喜欢的方向。如果看过觉得和你的阅读口味相近，再保持关注会更合适。

## 这个 RSS 有什么不同

它和市面上常见的公众号 RSS 工具有三点本质区别：

- **不是爬虫**：不会把你关注的所有公众号里的软文、硬广也一并推送出来，只输出我读过并筛选的内容
- **聚合高质量而非全量**：不需要逐个公众号去筛选订阅，省去在一堆广告推送中才看到一篇干货的过程
- **连文章内广告也会清理**：AI 辅助识别并移除正文内嵌的广告段落，阅读过程不会被突然插入的广告打断

RSS 每天凌晨 3 点更新，更新量取决于我当天的阅读量，节假日可能断更。

## 我为什么做这个项目

我平时会读很多公众号，但一个很明显的问题是：

很多号更新很勤，真正值得读的内容往往只有少数几篇，中间还常常夹着软广、硬广，或者会打断阅读节奏的推广段落。

现有不少公众号 RSS 工具更偏向“抓到什么就推什么”，很难把阅读体验里最烦人的那部分处理掉。这个项目就是为了解决这个问题：

- 尽量把更值得读的文章汇聚起来
- 尽量减少广告和无关内容对阅读的打扰
- 让公众号阅读回到更沉静、更连续的状态

目前项目采用的是 AI 与人工配合的方式：

- 用 AI 辅助做内容分析、质量筛选和 Markdown 整理
- 结合人工审核，尽量把广告干扰降下来
- 最终输出一条更适合订阅和长期阅读的 RSS

## 适合谁订阅

如果你希望看到的是：

- 更偏个人筛选而不是全量灌入的公众号阅读流
- 尽量减少广告干扰的订阅体验
- 可以顺手了解我长期阅读兴趣的一组样本

那这个 RSS 可能适合你。

如果你想要的是某个公众号的完整历史更新，这个项目就不是按那个目标设计的。

## 仓库说明

这个仓库**不是项目源码**，只是落地页与投稿入口。

可以直接在 [Issue](https://github.com/veesong/wxa-reader/issues) 中投稿公众号文章链接，AI 会自动处理并评分，评分太低的不会进入后续流程，质量达标后由我审核通过并进一步 AI 去广告处理，最终发布到 RSS。

如果有想推荐的高质量公众号，也欢迎在 Issue 中提出。我关注后日常阅读，觉得不错的文章也会投到这个 RSS 里。
