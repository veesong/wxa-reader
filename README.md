# 公众号阅读

这是一个围绕我个人日常阅读习惯搭建的公众号阅读项目。

平时看到值得保存或值得反复阅读的公众号文章，我会把它们投稿到这个系统里。项目会经过抓取、整理、筛选和审核，最后以 RSS 的形式输出，方便在更安静的阅读环境里继续看。

如果你想先看看我长期都在读什么、判断彼此的关注点和阅读口味是否接近，再决定要不要订阅，建议先看归档站。

## 这个 RSS 有什么不同

它不是常见那种“按公众号全量搬运”的 RSS。

这里输出的是我最近阅读并通过整理的文章流，目前只保留最近 10 篇，适合当作一个轻量的、不断更新的阅读入口，而不是完整的公众号镜像。

如果你更关心历史内容，可以去归档站查看已经整理过的文章集合。

## 我为什么做这个项目

我平时会读很多公众号，但一个很明显的问题是：

很多号更新很勤，真正值得读的内容往往只有少数几篇，中间还常常夹着软广、硬广，或者会打断阅读节奏的推广段落。

现有不少公众号 RSS 工具更偏向“抓到什么就推什么”，很难把阅读体验里最烦人的那部分处理掉。这个项目就是为了解决这个问题：

- 尽量把更值得读的文章汇聚起来
- 尽量减少广告和无关内容对阅读的打扰
- 让公众号阅读回到更沉静、更连续的状态

目前项目采用的是 AI 与人工配合的方式：

- 用 AI 辅助做内容分析、质量筛选和 Markdown 整理
- 结合人工审核，尽量把广告干扰降下来
- 最终输出一条更适合订阅和长期阅读的 RSS

## 关于归档站

归档站收录的是历史上已经整理完成的文章，适合：

- 先浏览我整体的阅读偏好
- 回看过去读过的内容
- 不想只看最近 10 篇，而是想看更完整的历史样本

为避免不必要的麻烦，归档站目前对大陆地区访问做了限制；如果你无法直接打开，可能需要使用可访问外网的网络环境。

## 适合谁订阅

如果你希望看到的是：

- 更偏个人筛选而不是全量灌入的公众号阅读流
- 尽量减少广告干扰的订阅体验
- 可以顺手了解我长期阅读兴趣的一组样本

那这个 RSS 可能适合你。

如果你想要的是某个公众号的完整历史更新，这个项目就不是按那个目标设计的。

## 仓库说明

这个仓库还会承接公众号投稿，投稿在 issue 中的文章地址会先经过 AI 评分，质量达标的我再审核通过并进一步 AI 去广告处理生成 RSS。

当然，如果你有什么推荐的公众号也可以提到 issue 中，我可能会关注加入日常的阅读列表，AI 评分高的文章一般都会加入到阅读归档中。
